**Architecture des Tables (Schéma Public) :**

1.  **`profiles`** : Utilisateurs de l'application.
    * `id` (uuid, PK) : Identifiant unique.
    * `email` (text) : Adresse de connexion.
    * `role` (text) : 'admin' ou 'user'.
2.  **`installations`** : Sites physiques (maisons/bâtiments).
    * `id` (uuid, PK).
    * `user_id` (uuid, FK) ➔ `profiles.id`.
    * `nom` (text) : Nom du site (ex: "Installation Test").
3.  **`capteurs`** : Matériel physique installé.
    * `id` (uuid, PK).
    * `installation_id` (uuid, FK) ➔ `installations.id`.
    * `type` (text) : Catégorie (ex: "meteo").
    * `nom` (text) : Nom personnalisé (ex: "Capteur Meteo 1").
4.  **`types_mesure`** : Dictionnaire des grandeurs physiques.
    * `id` (int2, PK) : 1=co2, 2=humidite, 3=temperature, 4=presence.
    * `code` (text) : Identifiant court.
    * `unite` (text) : Unité de mesure (ppm, %, degC, bool).
5.  **`mesures`** : Données temps réel envoyées par les capteurs.
    * `id` (int8, PK).
    * `capteur_id` (uuid, FK) ➔ `capteurs.id`.
    * `type_mesure_id` (int2, FK) ➔ `types_mesure.id`.
    * `value` (int8) : La donnée brute.
6.  **`choix_auto`** : Registre des modes de pilotage choisis.
    * `id` (int8, PK).
    * `choix` (text) : Mode actif (ex: "electric").
7.  **`alertes`** : Journal des anomalies.
    * `id` (int8, PK).
    * `capteur_id` (uuid, FK) ➔ `capteurs.id`.
    * `message` (text) : Description de l'alerte.
8.  **`notes_user`** : Commentaires libres.
    * `user_id` (uuid, FK) ➔ `profiles.id`.

**Logique de Jointure (Crucial pour le code) :**
* Pour obtenir la valeur d'une température pour un utilisateur :
